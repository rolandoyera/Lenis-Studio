# Alerts

The feedback surfaces used across the app — Sonner toasts and Badges — gathered
in one place. Both are now standardized: every usage pulls the shared component
from `components/ui` with no custom styling layered on top. These are the
canonical surfaces and **should not change without approval** — don't restyle a
toast or hand-roll badge colors; reach for an existing variant instead.

## Toasts

Built on [Sonner](https://sonner.emilkowal.ski/), configured once in
`components/ui/sonner.tsx`. This configuration is standard and shouldn't change
without approval. Trigger them to see the live styling:

<ToastGallery />

```ts
import { toast } from "sonner";

toast.success("Changes saved.");
toast.error("Couldn't load analytics", {
  description: "The request timed out. Try again in a moment.",
});
```

## Badges

From `components/ui/badge.tsx`. These are the sanctioned variants — the complete
set, used as-is with no custom styling. KPI trend deltas use `trendingUp` and
`trendingDown`. This set is standard and shouldn't change without
approval; reach for an existing variant rather than hand-rolling colors.
